# 更新日志

## [v0.2.3] - 2026-03-12

### 修复
- **修复浏览器响应超时问题**：增强消息计数检测逻辑，添加多个DOM选择器，使用Counter统计返回最常见计数
- **修复响应提取失败问题**：添加强制提取机制，30秒后强制尝试提取最新响应，避免无限等待
- **修复消息队列阻塞问题**：从串行处理改为并行处理，最多同时处理3个请求，避免后续请求被阻塞
- **修复日志错误影响响应问题**：添加try-catch包装logger.log_api_response，避免日志写入失败导致响应中断

### 改进
- **增强系统提示词**：
  - 添加【你的能力】部分，明确说明工具调用能力
  - 添加【强制规则】，确保模型在提及工具调用时使用正确格式
  - 在用户消息结尾添加工具调用提醒，避免模型遗忘
- **优化响应检测速度**：
  - 降低稳定性阈值，从>3改为>=2
  - 降低检测间隔，从0.5秒改为0.3秒
  - 降低消息计数稳定检测，从<3改为<1
- **改进消息队列性能**：
  - 并行处理多个请求，提升队列处理效率3倍
  - 动态调度，任务完成后立即启动新任务
  - 单个任务失败不影响其他任务

### 新增
- 强制提取机制：30秒后强制尝试提取最新响应
- 并行队列处理：最多同时处理3个请求
- 工具调用提醒：每个用户消息结尾自动添加工具调用提示
- **精简系统提示词**：优化JSON示例格式，减少不必要的字段描述，降低token消耗
- **DeepSeek自动新建对话**：每5次API请求后自动点击"开始新对话"，避免上下文过长导致性能下降

## [v0.2.2] - 2026-03-10

### 修复
- 修复工具调用响应解析问题：嵌套JSON arguments格式处理（使用括号匹配算法替代正则表达式）
- 修复重复响应问题：改进响应元素追踪逻辑，记录初始响应内容并检测内容变化
- 修复响应检测问题：当响应元素数量未增加时，检测内容是否变化
- 修复系统提示词合并问题：正确合并基础提示词与用户自定义人格设定

### 改进
- 增强系统提示词：添加环境信息（Windows命令）、语言规则（中文优先）、工具调用禁令
- 改进响应等待逻辑：支持检测响应元素内容变化
- 改进日志记录：记录初始响应内容和内容变化事件

### 新增
- 支持自定义系统提示词和人格设定合并
- 响应内容变化检测机制
